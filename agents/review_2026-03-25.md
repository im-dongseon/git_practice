# Git 강의 자료 재리뷰

작성일: 2026-03-25

## 1. 이번 재리뷰의 관점

이번 리뷰는 기존 강의 자료를 한 차례 개선한 뒤, **업데이트된 강의가 실제로 더 자연스럽고 일관된 학습 경험을 주는지**를 다시 점검하는 목적에서 작성했다.

특히 다음 세 가지를 중심으로 살폈다.

- 강의 간 맥락 연결이 자연스러운가
- 현재 강의 순서가 학습 흐름상 적절한가
- 전체적인 명령어 사용이 지금의 Git 학습 문맥에 어울리는가

결론부터 말하면, 이번 업데이트 이후 강의의 완성도는 분명히 올라갔다. 이전에는 부분적으로 보이던 명령어 혼용과 설명 공백이 많이 줄었고, 강의 전체가 **“Git을 처음 배우는 사람이 실무형 흐름을 따라가는 과정”** 으로 더 선명해졌다.

---

## 2. 전체 총평

현재 강의는 입문~초중급 수강생에게 적합한 **실습 중심 Git 협업 강의**로 꽤 안정적인 구조를 갖추고 있다.

특히 이번 수정으로 인해 다음 세 가지가 크게 좋아졌다.

- `git switch` / `git restore` 중심으로 명령어 체계가 정리됨
- `02_git.md`에서 Git 내부 구조를 먼저 설명해 뒤 문서 이해가 쉬워짐
- `04_conflict.md`, `05_rebase_squash.md`, `06_reset_revert.md`에 안전성 설명이 추가되어 실무 감각이 강화됨

즉, 이전에는 “좋은 실습 모음”에 가까웠다면, 지금은 **학습 순서와 개념 연결이 좀 더 설계된 강의 자료**에 가까워졌다.

---

## 3. 강의 간 맥락 연결 평가

### 전반 평가

강의 간 연결은 이전보다 자연스러워졌다.

현재 흐름은 아래처럼 읽힌다.

1. `01_quick_setup.md`에서 실습 환경 준비
2. `02_git.md`에서 Git의 역할과 내부 구조 이해
3. `03_start_new_feature.md`에서 기본 작업 흐름 실습
4. `04_conflict.md`에서 협업 중 충돌 처리
5. `05_rebase_squash.md`에서 이력 정리와 충돌 예방
6. `06_reset_revert.md`에서 복구 전략 학습
7. `98_tip.md`, `99_etc.md`에서 실무 팁과 문화 보완

이 구조는 “시작 → 작업 → 충돌 → 정리 → 복구 → 운영 습관”의 흐름을 형성하고 있어서, 학습자 입장에서 맥락이 잘 이어진다.

### 특히 좋아진 연결 지점

#### `02_git.md` → `03_start_new_feature.md`

이전보다 훨씬 좋아졌다.

`02_git.md`에 추가된 **Working Directory / Staging Area / Local Repository** 설명은 이후 `git restore`, `git add`, `git reset`, `git revert`, `git stash`를 이해하는 데 직접적인 발판이 된다. 이제는 `03_start_new_feature.md`에서 `git restore`가 나와도, 단순 암기가 아니라 “작업 영역을 되돌리는 명령”으로 받아들이기 쉬워졌다.

#### `03_start_new_feature.md` → `04_conflict.md`

기능 브랜치를 만들고 병합하는 흐름 뒤에 바로 conflict를 배치한 것은 여전히 적절하다.

특히 `03_start_new_feature.md`에서 merge를 “충돌 가능성이 있지만 해결 가능하다”는 방향으로 수정한 덕분에, `04_conflict.md`로 넘어가는 연결이 더 자연스러워졌다. 이전보다 “merge는 무서운 것”이 아니라 “협업 중 발생할 수 있는 상황”으로 읽힌다.

#### `04_conflict.md` → `05_rebase_squash.md`

이 연결도 좋다.

충돌을 직접 겪은 뒤, “어떻게 하면 충돌을 줄이고 이력을 깔끔하게 관리할까?”로 넘어가는 흐름은 교육적으로 설득력이 있다. 단순히 rebase를 기술 기능으로 소개하지 않고, conflict 맥락 뒤에 배치한 점이 좋다.

#### `05_rebase_squash.md` → `06_reset_revert.md`

이 순서도 무난하다.

먼저 **이력을 정리하는 방법**을 배우고, 그 다음 **실수했을 때 되돌리는 방법**을 배우는 흐름이다. 둘 다 Git 이력 조작과 관련된 주제라서 학습자의 머릿속에서 묶이기 쉽다.

### 아직 남아 있는 연결상의 작은 아쉬움

#### `01_quick_setup.md`와 본편 연결

`01_quick_setup.md`는 환경 준비 문서로 잘 작동하지만, 본편과의 연결 문장이 조금 더 있으면 좋다.

예를 들어 마지막에 아래 같은 문장이 있으면 흐름이 더 매끄러워진다.

- “실습 환경 준비가 끝났다면 다음 문서에서 Git과 GitHub의 차이부터 살펴봅니다.”

현재도 이해에는 문제 없지만, 강의형 자료로서는 이런 연결 문장이 있으면 더 친절하다.

#### `98_tip.md`의 위치 설명

`98_tip.md`는 부록처럼 잘 분리되어 있지만, “본편 학습 후 읽으면 좋은 실무 보조 기능”이라는 안내가 앞에 한 줄 있으면 더 좋다. 지금은 독립 문서로는 괜찮지만, 커리큘럼 안에서의 위치 설명은 약간 약하다.

## 4. 강의 순서 변경 필요성 평가

### 결론

**지금 당장 순서를 바꿔야 할 필요는 크지 않다.**

현재 순서는 충분히 자연스럽고, 특히 입문자 기준에서는 다음 장점이 있다.

- 너무 이론적으로 시작하지 않음
- 너무 어려운 주제(rebase, reset)를 초반에 던지지 않음
- 충돌과 복구를 기능 개발 흐름 뒤에 배치해 맥락을 살림

### 현재 순서가 적절한 이유

#### `02_git.md`가 너무 늦지 않다

기본 개념과 내부 구조 설명을 `03_start_new_feature.md`보다 먼저 두는 것은 적절하다. 지금 추가된 내부 구조 섹션 덕분에, 이후 문서가 훨씬 잘 읽힌다.

#### `04_conflict.md`가 `05_rebase_squash.md`보다 먼저 오는 것이 맞다

입문자는 먼저 충돌이라는 현상을 직접 보고, 그 뒤에 rebase를 배우는 편이 이해하기 쉽다. rebase를 먼저 배우면 “왜 이걸 하지?”가 약해질 수 있다.

#### `06_reset_revert.md`는 뒤쪽에 두는 것이 맞다

reset/revert는 위험도와 개념 난도가 상대적으로 높다. 초반에 두면 학습자에게 “Git은 되돌리는 명령이 너무 많은 어려운 도구”처럼 느껴질 수 있다. 지금처럼 뒤에서 정리하는 편이 심리적 부담이 덜하다.

### 선택적으로 고려할 수 있는 순서 보정

필수는 아니지만 아래 정도는 고려할 수 있다.

#### 옵션 1: `98_tip.md` 일부를 `03_start_new_feature.md` 뒤로 약하게 연결

예를 들어 stash는 실제 개발 흐름과 밀접하므로, `03_start_new_feature.md` 말미에 “작업 도중 브랜치를 잠깐 바꿔야 할 때는 `98_tip.md`의 stash도 참고” 정도의 연결만 있어도 좋다.

문서 순서를 바꿀 필요는 없고, 교차 참조 정도면 충분하다.

#### 옵션 2: `99_etc.md`를 완전 부록임을 더 명확히 표시

현재도 사실상 부록이지만, 제목이나 첫 문장에서 “참고/부록” 성격을 조금 더 드러내면 학습자가 본편과 부록을 구분하기 쉬워진다.

## 5. 명령어 사용 일관성 평가

### 가장 크게 개선된 부분

이번 수정에서 가장 좋은 변화는 **`git checkout`의 다중 역할을 줄이고 `git switch`/`git restore` 기준으로 정리한 것**이다.

이 변경은 단순한 최신화가 아니라, 교육 품질 측면에서 의미가 크다.

- 브랜치 이동은 `git switch`
- 파일 되돌리기는 `git restore`
- 과거 방식 `git checkout`은 참고용으로만 유지

이 구조 덕분에 초보자가 “checkout이 브랜치도 바꾸고 파일도 되돌린다”는 혼란에 덜 빠지게 된다.

### 문서별 명령어 적합성

#### `03_start_new_feature.md`

전반적으로 잘 정리됐다.

- 브랜치 생성/이동: `git switch`, `git switch -c`
- 파일 복구: `git restore`
- 구방식 예시: `git checkout -b`

학습자에게 최신 방식과 과거 방식을 함께 보여주되, 기준은 명확히 최신 방식으로 둔 점이 좋다.

#### `04_conflict.md`

실습 단계에서 `git switch`를 사용한 것도 일관성이 좋다.

다만 맨 처음 충돌 발생 예시의 상단 코드 블록은 여전히 `git merge main`만 단독으로 제시되어 있는데, 초보자에게는 “어느 브랜치에서 실행하는가”가 다소 생략되어 보일 수 있다. 큰 문제는 아니지만, 상황 설명을 한 줄 더 보강하면 더 좋다.

#### `05_rebase_squash.md`

`git switch`로 예시를 바꾼 것은 적절하다. 또한 rebase를 history rewrite로 설명한 것도 좋다.

다만 `git rebase main` 예시에서는 “먼저 feature 브랜치로 이동한 뒤 실행”이므로 문제 없지만, 초보자를 위해 “현재 브랜치가 대상 브랜치가 아니라 내 작업 브랜치여야 한다”는 문장이 있으면 더 친절하다.

#### `06_reset_revert.md`

가장 많이 좋아진 문서다.

이전에는 `restore`와 `checkout`이 충돌하던 지점이 있었는데, 이제는 신규 방식과 구방식을 비교해서 보여 주기 때문에 학습 혼선이 크게 줄었다. 강의 전체 기준에서도 이 문서가 명령어 표준을 정리해 주는 역할을 한다.

### 아직 남아 있는 작은 명령어 이슈

#### `98_tip.md`의 `git stash save`

여기만 보면 살짝 예전 스타일이 남아 있다.

`git stash save "..."`는 예전 문맥에서는 흔했지만, 현재는 보통 아래처럼 더 단순하게 쓰는 경우가 많다.

```bash
git stash
git stash push -m "작업 내용 설명"
```

반드시 수정해야 할 정도는 아니지만, 이번 강의가 `switch`/`restore` 중심의 현대적 흐름으로 정리된 만큼 이 부분도 같은 톤으로 맞추면 더 좋다.

#### `git push --force-with-lease` 설명 보강 가능

`98_tip.md`에서 force push보다 안전한 방식으로 잘 수정되었지만, 왜 더 안전한지 한 문장 덧붙이면 초보자 이해가 더 좋아진다.

예:

- “원격에 내가 모르는 새 변경이 생겼으면 덮어쓰지 않도록 한 번 더 확인해 주는 방식”

## 6. 업데이트 이후 가장 좋아진 문서

### 1위: `06_reset_revert.md`

이번 업데이트에서 가장 개선 폭이 큰 문서다.

- 현재 권장 방식과 구방식 비교
- `git switch` / `git restore` 분리 이유 설명
- `reset --hard` 경고 강화
- 실습 저장소 기준 안내

이 문서는 지금 강의 전체의 “명령어 정리 기준점” 역할까지 하게 됐다.

### 2위: `02_git.md`

내부 구조 설명이 들어가면서 단순 개론 문서가 아니라, 뒤 강의를 받쳐주는 개념 허브 역할을 하게 됐다. 짧지만 효과가 큰 수정이다.

### 3위: `04_conflict.md`

충돌 해결 후 `git status`, `git log --graph --oneline --all`로 결과를 확인하게 만든 점이 특히 좋다. 이제는 “충돌 해결”이 단순 편집이 아니라 “해결 후 검증까지”라는 흐름으로 읽힌다.

## 7. 아직 남아 있는 개선 후보

필수 수정은 아니지만, 다음 정도는 추가로 고려할 수 있다.

### 1. `01_quick_setup.md`의 인증 파트 현대화

여전히 `Personal access tokens (classic)` 중심 안내라 약간 옛 느낌이 있다. SSH 방식이나 최신 토큰 사용 흐름까지 간단히 언급하면 더 좋다.

### 2. `98_tip.md`의 stash 명령 최신화

`git stash save` 대신 `git stash push -m` 또는 `git stash` 중심으로 정리하면 현재 강의 톤과 더 잘 맞는다.

### 3. 본편 문서 간 짧은 교차 참조 추가

예를 들어 아래 같은 연결 문장이 있으면 더 매끄럽다.

- `03_start_new_feature.md` 말미: “충돌이 나는 경우는 다음 문서에서 직접 실습합니다.”
- `04_conflict.md` 말미: “충돌을 줄이고 커밋을 정리하는 방법은 다음 문서의 rebase/squash를 참고합니다.”
- `05_rebase_squash.md` 말미: “잘못된 변경을 되돌리는 방법은 `06_reset_revert.md`에서 이어집니다.”

이런 연결은 문서 순서를 바꾸지 않고도 흐름 체감을 더 좋게 만든다.

## 8. 최종 판단

업데이트된 현재 강의는 **강의 간 맥락 연결이 전보다 자연스럽고, 강의 순서도 유지할 가치가 충분하며, 명령어 사용 역시 전반적으로 잘 어울리는 상태**다.

특히 `git switch` / `git restore` 중심 정리는 교육적으로 매우 적절했고, `02_git.md`의 내부 구조 설명 추가는 전체 강의의 이해도를 끌어올리는 좋은 수정이었다.

따라서 지금 상태의 판단은 아래와 같다.

- **강의 간 맥락 연결**: 좋음
- **강의 순서 변경 필요성**: 높지 않음
- **전체 명령어 사용 적절성**: 전반적으로 좋음
- **남은 보완 포인트**: `98_tip.md`의 stash 표현 현대화, `01_quick_setup.md`의 인증 최신화, 문서 간 짧은 교차 참조 추가

한 줄로 정리하면, 이 강의는 이제 **입문자가 Git을 실무 흐름 안에서 배우기에 꽤 안정적이고 설계된 자료**가 되었으며, 남은 과제는 구조 변경보다 마감 품질을 높이는 수준의 세부 다듬기라고 볼 수 있다.
