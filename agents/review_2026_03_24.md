### 03_start_new_feature.md — 새로운 기능 개발 시작 ✅

**내용 요약**  
Fork → 브랜치 생성 → 커밋/푸시 → PR/Merge 전체 흐름을 커버. merge vs cherry-pick 비교.

**장점**
- 실습 흐름이 가장 완성도 높음
- `git log --graph --all --oneline` 등 실용적인 명령어 포함
- merge와 cherry-pick을 이미지와 함께 비교 설명해 이해도 향상
- Branch를 "평행세계"로 비유한 표현이 직관적

**개선 필요**
- `git branch feature/add-profile && git checkout feature/add-profile` 를 `git checkout -b feature/add-profile` 로 단축 표현도 소개하면 좋음
- merge 시 fast-forward와 no-fast-forward 차이 설명 추가 고려
- PR(Pull Request)에 대한 별도 설명 또는 링크 부재

### 04_conflict.md — 충돌(Conflict) 해결 ⚠️

**내용 요약**  
두 명이 같은 파일을 수정할 때 발생하는 충돌, conflict 표시 확인, abort 방법.

**장점**
- conflict 마커(`<<<<<<<`, `=======`, `>>>>>>>`)를 직접 언급
- merge --abort, cherry-pick --abort 양쪽 모두 다룸

**개선 필요**
- ❌ **내용이 지나치게 짧음** (37줄 중 실제 내용 20줄 미만)
- conflict 발생 후 **직접 편집해서 해결하는 과정** 설명이 없음
  - 예: 에디터에서 충돌 영역을 선택하고 저장 후 `git add` → `git commit` 하는 흐름
- VSCode의 "Accept Current Change / Accept Incoming Change" UI 활용 방법 추가 권장
- `git diff` 명령어로 충돌 내용 미리 확인하는 방법 추가 권장

### 05_rebase_squash.md — Rebase / Squash ❌

**내용 요약**  
conflict를 줄이는 방법으로 rebase 언급, interactive rebase와 squash 명령어 제시.

**장점**
- rebase와 squash 개념을 이미지와 함께 소개

**개선 필요**
- ❌ **전체 분량이 매우 부족** (21줄)
- `git rebase -i HEAD~3` 명령 후 나타나는 에디터 화면 설명이 없음 (pick, squash, fixup 옵션 등)
- `git merge --squash`와 `git rebase -i`의 차이 설명 없음
- rebase 도중 충돌 발생 시 처리 방법(`git rebase --continue`, `--abort`, `--skip`) 누락
- **실습 예제가 없음**: 단계별 실습 시나리오 추가 강력 권장

### 07_tip.mkd — git stash / commit --amend ⚠️

**내용 요약**  
임시 저장(stash)과 최근 커밋 수정(amend) 두 가지 실용 팁.

**장점**
- git stash 명령어를 용도별로 잘 정리 (save, list, pop, apply, drop, clear)
- amend의 주의사항(force push 필요)을 ❗ 아이콘과 함께 강조
- 실용적인 사용 시나리오 목록 포함

**개선 필요**
- ❌ **파일 확장자 오류**: `.mkd` → `.md` 로 통일 필요
- `git stash -u`(untracked 파일까지 stash) 옵션 미언급
- `git stash branch <branch-name>` 명령어(stash를 새 브랜치로 복원) 미언급
- force push의 위험성(팀원의 이력 손상 가능성) 좀 더 강조 필요
